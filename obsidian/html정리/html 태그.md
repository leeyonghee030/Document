I. 텍스트 및 구조 관련 태그
1. 블록 요소 (Block Elements): 구조 및 단락
특징: 항상 새 줄에서 시작하며, 너비 전체를 차지합니다.

태그	용도 및 의미	메모
<p>	단락 (Paragraph)	텍스트의 기본 블록 단위.
<h1> ~ <h6>	제목 (Heading)	**<h1>**이 최고 중요도.
<div>	구획 (Division)	CSS/JS를 위한 범용 컨테이너 (의미 없음).
<blockquote>	긴 인용 블록	다른 출처에서 인용한 긴 텍스트.
<pre>	미리 서식이 지정된 텍스트	공백/줄 바꿈을 그대로 유지 (코드, ASCII).
 
2. 인 라인 요소 (Inline Elements): 강조 및 꾸밈
특징: 줄의 흐름을 따라 옆으로 배치되며, 내용물 크기 만큼의 너비만 가집니다.

태그	용도 및 의미	메모
<strong>	강력한 중요성	굵게 표시.
<em>	강조 (Emphasis)	기울임 꼴 표시.
<mark>	참조할 만한 부분	<mark>형광펜</mark> 효과.
<code>	컴퓨터 코드 조각	짧은 코드 표시.
<span>	스타일링 컨테이너	CSS/JS 적용을 위한 가장 기본적인 인 라인 요소.
<a>	앵커 (Anchor)	하이퍼 링크 생성.
<abbr>	약어 (Abbreviation)	title 속성으로 전체 이름 제공 가능.
<cite>	작품의 제목	책, 논문, 노래 등의 제목 표시.

3. 기타 제어 요소
태그	용도 및 의미	메모
<br>	줄 바꿈 (Line Break)	문단 내 강제 줄 바꿈. (빈 태그)
<hr>	수평선 (Horizontal Rule)	주제 별 구분을 위한 가로선. (빈 태그)

II. 폼 (Form) 및 입력 관련 태그
1. <input> 태그의 주요 type 속성
type 속성 값	용도 및 특징	메모
in put안에 placeholder 태그 이용시 인풋안에 텍스트 가능
text	기본적인 한 줄 텍스트 입력 란.	이름, 주소 등 일반 텍스트 입력.
password	입력 문자를 가려서 표시.	비밀번호 입력 시 사용.
email	이메일 유효성 검사 기능 포함.	모바일에서 이메일 전용 키보드 호출.
number	숫자만 입력 가능. (스피너 표시)	수량, 나이 등 숫자 입력.
search	검색어 입력 최적화 (브라우저에 따라 'X' 버튼 표시).	검색 창에 주로 사용.
checkbox	다중 선택이 가능한 사각형 체크 박스.	여러 항목을 동시에 선택할 때.
radio	단일 선택만 가능한 원형 버튼.	성별, 옵션 등 하나만 선택할 때.
submit	폼 데이터를 서버로 제출하는 버튼.	폼 전송의 핵심 버튼.
file	사용자가 파일을 선택하여 업로드.	이미지, 문서 등 파일 첨부.
hidden	사용자에게 보이지 않는 숨겨진 입력란.	서버에 전달할 데이터(ID 등)를 숨겨둘 때.

2. <button> 태그의 type 속성
<button>은 텍스트 외에 HTML 콘텐츠를 버튼 내부에 포함할 수 있습니다.

type 속성값	용도 및 특징	메모
submit	폼 제출 버튼. (기본값)	회원가입, 로그인 등 폼 데이터 전송.
button	일반 버튼. (기본 기능 없음)	JavaScript 코드를 실행할 때 사용.
reset	폼 초기화 버튼.	폼 내부의 모든 입력값을 초기화.

3. 드롭다운 메뉴 (<form>, <select>, <option>)
이 세 태그는 드롭다운 메뉴를 만들 때 계층적으로 사용됩니다.

<form>: 데이터를 전송할 전체 영역을 정의하는 컨테이너.

<select>: 사용자에게 드롭다운 메뉴를 제공하는 실제 입력 필드.

<option>: <select> 태그 내부에 위치하여 사용자가 선택할 수 있는 **개별 항목(선택지)**을 정의.

III. 기타 주요 구조 태그
1. <img> 태그 (이미지)
<img> 태그는 type 속성이 없으며, 이미지를 삽입하는 단일 목적을 가집니다.

속성	용도 및 의미	메모
src	필수! 표시해야 할 이미지 파일의 경로를 지정.	Source의 약자.
alt	필수! 이미지를 표시할 수 없을 때 나타낼 대체 텍스트 지정.	웹 접근성 및 SEO에 중요.

2. <table> 태그 (표)
<table> 태그는 데이터를 행(row)과 열(column)로 구조화하는 데 사용됩니다.

A. 표를 구성하는 필수 요소
태그	용도 및 의미	메모
<tr>	표의 한 행 (Table Row)을 정의.	모든 셀을 감싸는 컨테이너.
<td>	표의 일반적인 데이터 셀 (Table Data)을 정의.	<tr> 내부에 위치.
<th>	표의 제목 셀 (Table Header)을 정의.	행 또는 열의 제목으로 사용.

B. 표의 구조 및 의미 요소
태그	용도 및 의미	메모
<caption>	표의 제목 또는 설명을 정의.	<table> 바로 다음에 위치.
<thead>	표의 머리글 영역을 그룹화.	(컬럼 제목)
<tbody>	표의 본문 데이터 영역을 그룹화.	(실제 데이터)
<tfoot>	표의 바닥글 영역을 그룹화.	(합계, 평균 등)

C. 셀 병합 속성
속성	용도 및 의미	메모
colspan	셀을 가로 방향으로 병합.	열(Column) 병합.
rowspan	셀을 세로 방향으로 병합.	행(Row) 병합.
