# ▦ CSS Grid 레이아웃 완벽 정리

Flexbox가 1차원(한 방향)이라면, Grid는 **2차원(가로+세로)** 레이아웃을 다룹니다.
웹페이지 전체의 큰 구조를 잡을 때 매우 유용합니다.

## ⚡️ 한눈에 보는 요약표

| 속성                        | 적용 대상 | 역할                 | 비유 (이해 쏙쏙) |
| :------------------------ | :---- | :----------------- | :--------- |
| **display: grid**         | 부모    | 그리드 선언             | 바둑판 깔기     |
| **grid-template-columns** | 부모    | **가로(열)** 크기/개수 정의 | 기둥 세우기     |
| **grid-template-rows**    | 부모    | **세로(행)** 크기/개수 정의 | 층 나누기      |
| **gap**                   | 부모    | 아이템 사이 간격          | 골목길 만들기    |
| **grid-column**           | 자식    | 가로 영역 차지하기         | 땅따먹기 (가로)  |
| **grid-row**              | 자식    | 세로 영역 차지하기         | 땅따먹기 (세로)  |

---

## 1. 부모 요소(Container) 속성: 바둑판 만들기
> 전체 레이아웃의 행(Row)과 열(Column) 구조를 만듭니다.

### 📌 `grid-template-columns` (열 배치)
가로 기둥을 몇 개 만들고, 각각의 너비를 얼마로 할지 정합니다.
* **픽셀 고정:** `200px 200px 500px` (3개 기둥)
* **비율(`fr`):** `1fr 1fr 1fr` (화면을 3등분)
    * *`fr`은 fraction(비율)의 약자입니다.*
* **반복(`repeat`):** `repeat(4, 1fr)` (1fr짜리 기둥 4개 생성)



### 📌 `grid-template-rows` (행 배치)
세로 층을 어떻게 나눌지 정합니다. 사용법은 `columns`와 같습니다.
* 보통 높이는 내용물에 따라 자동 조절되도록 생략하거나 `auto`를 많이 씁니다.

### 📌 `gap` (간격)
셀과 셀 사이의 여백을 줍니다.
* `gap: 20px;` (가로세로 모두 20px 간격)
* `row-gap`, `column-gap`으로 따로 줄 수도 있습니다.

---

## 2. 자식 요소(Item) 속성: 땅따먹기
> 만들어진 바둑판 위에서 아이템이 몇 칸을 차지할지 결정합니다.
> **주의:** 칸 개수가 아니라 **'선(Line) 번호'**를 기준으로 합니다.



### 🔹 `grid-column` (가로 영역)
가로로 몇 번 줄부터 몇 번 줄까지 차지할지 정합니다.
* **문법:** `시작 번호 / 끝 번호`
* 예시: `grid-column: 1 / 3;` (1번 선에서 시작해서 3번 선까지 = **2칸 차지**)
* **단축키:** `grid-column: span 2;` (시작 위치 상관없이 그냥 **2칸 차지**)

### 🔹 `grid-row` (세로 영역)
세로로 몇 번 줄부터 몇 번 줄까지 차지할지 정합니다.
* 예시: `grid-row: 1 / 3;` (1번째 줄부터 3번째 줄까지 확장)

---

## 💡 Flexbox vs Grid 차이점 (언제 뭘 써요?)

| 구분       | Flexbox                    | Grid                       |
| :------- | :------------------------- | :------------------------- |
| **차원**   | **1차원** (한 줄 배치)           | **2차원** (면 배치)             |
| **주사용처** | 메뉴바, 카드 리스트 정렬, 작은 컴포넌트 내부 | 전체 웹페이지 레이아웃, 복잡한 신문 기사 배치 |
| **비유**   | 목걸이에 구슬 꿰기                 | 엑셀 표 만들기                   |
|          |                            |                            |
# 🚀 CSS Grid 실전 속성 & 코드 해석집

사용자가 자주 마주치는 Grid 코드 패턴과 실무에서 꼭 필요한 추가 속성들을 정리했습니다.

---

## 1. 부모(Container) 속성: 판 깔기 심화

### 📐 `grid-template-columns` (열/가로 기둥 정의)
> 다양한 단위를 섞어 쓸 수 있는 것이 Grid의 장점입니다.

* **`repeat(8, 12.5%)`**
    * **의미:** 12.5% 너비의 기둥을 8개 만듭니다. (12.5 * 8 = 100%)
    * **용도:** 8등분 그리드를 만들 때 씁니다.
* **`100px 3em 40%`**
    * **의미:** 1번 기둥 100px(고정), 2번 기둥 3글자 크기(가변), 3번 기둥 40%(비율).
    * **용도:** 고정 메뉴바 + 유동적인 본문을 섞을 때.
* **`1fr 5fr`**
    * **의미:** 남은 공간을 1:5 비율로 나눕니다.
    * **용도:** 사이드바(좁게) + 본문(넓게).

### 📐 `grid-template-rows` (행/세로 층 정의)
* **`20% 20% 20% 20% 20%`**
    * **의미:** 화면 높이를 정확히 5등분 합니다.

### ⚡️ `grid-template` (초강력 단축 속성)
> `rows`와 `columns`를 한 줄에 적는 끝판왕 속성입니다.

* **문법:** `grid-template: (행 정의) / (열 정의);`
* **예시:** `1fr 50px / 20% 1fr`
    * **행(Rows):** 첫 줄은 `1fr`, 둘째 줄은 `50px`
    * **/** (구분선)
    * **열(Columns):** 첫 칸은 `20%`, 둘째 칸은 `1fr`

---

## 2. 자식(Item) 속성: 위치 잡기 (땅따먹기)

### 📍 `grid-column` & `grid-row` (영역 지정)
* **`grid-column-start: 5;`**
    * 5번 세로선(Line)에서 시작합니다.
* **`grid-column-end: 4;`**
    * 4번 세로선에서 끝납니다. (보통은 `start`보다 큰 숫자를 쓰지만, 순서는 상관없습니다.)
* **`grid-column: 2 / 5;`**
    * **의미:** 2번 선부터 5번 선까지 차지합니다. (총 3칸)
    * **팁:** `start / end` 순서입니다.

### 📍 `grid-area` (한방에 영역 지정)
> `row-start`, `column-start`, `row-end`, `column-end` 4가지를 한 번에 씁니다.

* **문법:** `row-start / col-start / row-end / col-end`
* **암기법:** **"행열행열" (시작 / 시작 / 끝 / 끝)**
* **예시:** `grid-area: 1 / 2 / 4 / 6;`
    * 세로 1번 줄, 가로 2번 줄에서 **시작**해서
    * 세로 4번 줄, 가로 6번 줄에서 **끝**납니다.

### 🔄 `order` (순서 바꾸기)
* **`order: 1;`**
    * **의미:** HTML 작성 순서와 상관없이 시각적으로 뒤로 보냅니다. (기본값 0)
    * Flexbox와 사용법이 똑같습니다.

---

## 3. 알아두면 프로처럼 보이는 추가 속성들 ✨

### 🧲 정렬 (Flexbox랑 비슷해요!)
아이템을 셀(칸) 안에서 어떻게 배치할지 결정합니다.

| 속성 | 역할 | 비유 |
| :--- | :--- | :--- |
| **`justify-items`** | 가로 정렬 (왼쪽/오른쪽/중앙) | 셀 안에서 좌우로 움직이기 |
| **`align-items`** | 세로 정렬 (위/아래/중앙) | 셀 안에서 위아래로 움직이기 |
| **`place-items`** | 위 두 개를 한 번에 (`center center`) | 정가운데 꽂기 |

### 👻 `grid-auto-rows` / `grid-auto-columns` (암시적 그리드)
내가 `grid-template`으로 정의하지 않은, **예상보다 더 많은 아이템이 들어왔을 때** 그 아이템의 크기를 어떻게 할지 정합니다.

* **예시:** `grid-auto-rows: 100px;`
    * "내가 3줄만 만들었는데 아이템이 많아서 4번째 줄이 생기면? 걔는 자동으로 100px로 해줘."

### 📏 `minmax()` (최소 최대 크기)
반응형 웹에서 필수입니다.
* **예시:** `grid-template-columns: minmax(100px, 1fr);`
    * "아무리 작아져도 **100px**은 지켜줘. 근데 공간이 남으면 **1fr**로 쫙 늘어나도 돼."

---

## 4. 예제로 보는 `grid-area` 해석
```css
.item {
  /* 행 시작 / 열 시작 / 행 끝 / 열 끝 */
  grid-area: 1 / 2 / 4 / 6;
}

grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
