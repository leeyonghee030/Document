## 1. 자료형 (Data Type)

> **한 줄 정의:** "이 변수에 어떤 종류의 값을 넣을 수 있는지 정해주는 규칙"

- **필요성:** 컴퓨터는 모든 값을 똑같이 다루지 않으므로 숫자, 문자, 참/거짓을 구분하여 효율적으로 처리하기 위함입니다.
    

### 🔹 기본형 vs Wrapper 매핑 표

|**기본 자료형 (알맹이)**|**래퍼 클래스 (상자)**|**특징**|
|---|---|---|
|`byte`|`Byte`|아주 작은 정수|
|`short`|`Short`|짧은 정수|
|**`int`**|**`Integer`**|**가장 많이 쓰는 정수 (주의!)**|
|`long`|`Long`|아주 큰 정수 (끝에 `L` 붙임)|
|`float`|`Float`|소수 (끝에 `f` 붙임)|
|**`double`**|**`Double`**|**실무 표준 소수**|
|**`char`**|**`Character`**|**문자 1개 (주의!)**|
|`boolean`|`Boolean`|참(`true`) / 거짓(`false`)|

- **참조 자료형 (String):** 문자를 여러 개 저장하는 **문자열 객체**입니다. (기본 자료형 아님!)
    

---

## 2. 클래스 & 객체 (Class & Object)

> **한 줄 정의:** "객체를 만들기 위한 설계도"

### 🔹 클래스 vs 객체 vs 인스턴스

|**구분**|**클래스 (Class)**|**객체 (Object / Instance)**|
|---|---|---|
|**성격**|설계도, 틀, 개념|제품, 실제 물건, 실체|
|**위치**|소스 코드 파일 (.java)|컴퓨터의 메모리 (RAM)|
|**상태**|값이 정해져 있지 않음|구체적인 값을 가지고 있음|

- **객체(Object):** 모든 실체를 통틀어 부르는 포괄적인 이름.
    
- **인스턴스(Instance):** "특정 클래스로부터 만들어졌다"는 **관계**를 강조할 때 쓰는 말.
    

### 🔹 클래스의 3요소

클래스라는 설계도 안에는 딱 이 세 가지만 기억하면 됩니다.

### ① 변수 (필드 / 속성)

- **특징:** 객체의 **상태나 데이터**를 저장합니다. 클래스 전역에서 사용 가능합니다.
    
- **예시:** `String color;`, `int speed;`
    

### ② 메소드 (기능 / 동작)

- **특징:** 객체가 할 수 있는 **행동**을 정의합니다. 이름 뒤에 `()`가 붙습니다.
    
- **예시:** `void drive() { System.out.println("달린다"); }`
    

### ③ 생성자 (초기화)

- **특징:** `new`로 객체를 만들 때 자동 실행되며, **변수값을 초기 세팅**합니다. 클래스와 이름이 같고 반환형이 없습니다.
    
- **예시:** `public Car(String c) { this.color = c; }`

## 3. 메소드 (Method) 구성요소

> 클래스 내부의 **기능**. `접근제어자` + `반환형` + `이름` + `매개변수`로 구성됩니다.

- **접근 제어자:** `public`, `private` 등 (누가 쓸 수 있는지)
    
- **반환형 (Return Type):** 결과값의 종류
    
    - `int`: 숫자 반환
        
    - `String`: 글자 반환
        
    - **`void`**: **반환할 값이 없음 (실행만 하고 끝!)**
        
- **메소드 이름:** 기능의 이름 (예: `saveUser`)
    
- **매개변수 (Parameter):** 일을 하기 위해 외부에서 받는 **재료**
    
- **메소드 본문 (Body):** `{ }` 중괄호 안의 코드로, **실제 로직이 돌아가는 부분**입니다.

## 4. 캐스팅 (Casting) : 형변환

### 🔹 업캐스팅 (Upcasting)

- **정의:** 자식 객체를 부모 타입으로 보는 것 (`Animal a = new Dog();`)
    
- **목적:** 자식의 개성을 잠시 포기하고, **공통된 부모 타입으로 묶어 효율적으로 관리**하기 위함.
    

### 🔹 다운캐스팅 (Downcasting)

- **정의:** 부모 타입으로 보던 객체를 다시 자식 타입으로 되돌리는 것 (`Dog d = (Dog) a;`)
    
- **목적:** 부모 타입으로 묶어놨다가, **자식만의 고유 기능**이 다시 필요할 때 사용.
    

---

## 5. Wrapper 클래스

> **한 줄 정의:** "기본 자료형을 객체라는 상자에 담아 포장한 버전"

- **사용 이유:** 객체만 받는 곳(List 등)에 넣기 위해, 유용한 메소드 사용을 위해, `null`을 표현하기 위해.
    
- **박싱(Boxing):** 기본형 → 래퍼 (알맹이 → 상자)
    
- **언박싱(Unboxing):** 래퍼 → 기본형 (상자 → 알맹이)

## 6. 상속과 구현 (extends & implements)

- **`extends` (상속):** 부모 클래스의 특징을 그대로 물려받아 **확장**할 때 사용.
    
- **`implements` (구현):** 인터페이스에 적힌 **기능 규칙**을 실제 클래스에서 완성할 때 사용.
    
- **인터페이스 (Interface):** "이 클래스는 이런 기능을 반드시 가져야 한다"는 **규칙서**.


### 1.상속 (extends) : "유전적 계승"

**개념:** 이미 존재하는 클래스를 물려받아 기능을 **'확장'**하는 것입니다. (Is-a 관계: ~은 ~의 일종이다)

- **비유:** 부모님의 **'재산과 유전자'**를 물려받는 것.
    
- **더 좋은 예시:** `휴대폰` ➔ `스마트폰`
    
    - **부모(Phone):** 전화하기, 문자하기 (기본 기능)
        
    - **자식(SmartPhone):** 전화/문자는 기본으로 하되, **인터넷 하기, 앱 설치하기**를 추가로 가짐.
        

Java

```
class Phone {
    void call() { System.out.println("전화 중..."); }
}

// 스마트폰은 휴대폰의 일종이면서 기능을 확장함
class SmartPhone extends Phone {
    void internet() { System.out.println("인터넷 접속!"); }
}
```

---

### 2. 구현 (implements) : "기술 자격증"

**개념:** 클래스가 어떤 **'기능적 규칙'**을 반드시 따르겠다고 약속하는 것입니다. (Has-a capability: ~할 수 있는 능력이 있다)

- **비유:** 운전면허나 조리사 **'자격증'**을 따는 것. (사람이든 로봇이든 면허가 있으면 운전할 수 있음)
    
- **더 좋은 예시:** `날 수 있는(Flyable)` 인터페이스 ➔ `새` & `비행기`
    
    - **규칙(Flyable):** "이걸 쓰려면 무조건 `fly()` 기능을 만들어라!"
        
    - **클래스(Bird, Airplane):** 둘은 완전히 다른 종류지만, 둘 다 **'날 수 있다'**는 공통 규칙을 가짐.
        

Java

```
interface Flyable {
    void fly(); // "어떻게든 날아야 한다"는 규칙만 정함
}

class Bird implements Flyable {
    public void fly() { System.out.println("날개를 퍼덕이며 비행"); }
}

class Airplane implements Flyable {
    public void fly() { System.out.println("엔진을 가동하며 비행"); }
}
```

---

### 3. 한눈에 비교하기 (핵심 정리)

|**구분**|**상속 (extends)**|**구현 (implements)**|
|---|---|---|
|**비유**|**가족 (혈연)**|**자격증 (능력)**|
|**관계**|"스마트폰은 휴대폰의 일종이다"|"비행기는 날 수 있는 능력이 있다"|
|**목적**|부모의 코드를 **그대로 재사용**|정해진 규칙을 **강제로 완성**|
|**특징**|부모는 딱 한 명만 가능 (단일 상속)|자격증은 여러 개 딸 수 있음 (다중 구현)|

## 7. 추상화 (abstract)

> **한 줄 정의:** "완성되지 않은 추상적인 상태"

abstract class Animal {
    abstract void sound(); // 추상 메소드 (이름만 있음)
}

class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("야옹"); // 자식이 본문(Body)을 완성!
    }
}

- **추상 메소드:** `abstract void sound();` → "기능은 필요하지만, 내용은 자식이 정해라!"
    
- **왜 쓰나?** 부모는 공통 틀만 잡고, 세부 내용은 자식마다 다르게 만들게 강제하기 위해.

## 8. 접근 제어자 (Access Modifier)

- **`public`:** 어디서든 접근 가능.
    
- **`private`:** 오직 같은 클래스 안에서만 (데이터 보호).
    
- **`protected`:** 같은 패키지 + 상속받은 자식 클래스까지 허용.
    
- **`default` (아무것도 안 씀):** 같은 패키지 안에서만 허용.

## 9. 오버라이딩 vs 오버로딩

|**구분**|**오버라이딩 (Overriding)**|**오버로딩 (Overloading)**|
|---|---|---|
|**핵심**|**재정의** (올라타다)|**다중정의** (과적하다)|
|**상태**|부모의 메소드를 자식이 바꿔 씀|이름은 같지만 매개변수가 다른 메소드를 여러 개 만듦|
|**조건**|메소드 이름, 반환형, 매개변수 모두 동일|메소드 이름은 같고 **매개변수만 달라야 함**|
