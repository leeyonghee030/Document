## 1. Table 태그 구조 (Thead, Tbody)

표를 만들 때는 단순히 칸을 나누는 것이 아니라, 문서의 구조처럼 '머리-가슴-배'를 나누는 것이 가슴이 웅장해지는 정석적인 방법입니다.

- **`<table>`**: 표의 시작과 끝을 알립니다.
    
- **`<thead>`**: 표의 제목 행들을 모아두는 곳입니다.
    
- **`<tbody>`**: 표의 실제 데이터 본문이 들어갑니다.
    
- **`<tr>` (Table Row)**: 가로 한 줄을 의미합니다.
    
- **`<th>` (Table Header)**: 제목 칸 (두껍게 표시됨).
    
- **`<td>` (Table Data)**: 일반 데이터 칸.
  
  - **구조**: `caption`(표 제목), `thead`, `tbody`, `tfoot`으로 영역을 확실히 나누세요.
    
- **병합의 기술**:
    
    - `colspan`: 가로로 칸 합치기.
        
    - `rowspan`: 세로로 줄 합치기.
        
- **디자인 팁**: `border-collapse: collapse;`를 쓰면 촌스러운 이중 테두리가 사라집니다.
    

> [!tip] 왜 귀찮게 `thead`, `tbody`를 나누나요?
> 
> 표가 아주 길어질 때 브라우저가 제목(`thead`)만 고정하고 본문만 스크롤하게 하거나, 인쇄 시 페이지마다 제목을 반복해 주는 등 유익한 기능을 제공하기 때문입니다.

## 2. Form 태그와 Action / Method

`<form>`은 사용자 입력값을 감싸서 서버로 던져주는 역할을 합니다.

- **`action`**: 데이터를 보낼 목적지 주소입니다. (예: `/login.php`)
    
- **`method`**: 데이터를 보내는 "방식"입니다.
    
    - **`GET`**: 데이터를 주소창에 다 보여주면서 보냅니다. (검색어 전송 등 공개되어도 상관없는 데이터)
        
    - **`POST`**: 데이터를 편투 봉투에 넣어 숨겨서 보냅니다. (로그인 비밀번호, 개인정보 등 보안이 필요한 데이터)

## 3. 다양한 Input 타입 정리

`<input>`은 `type` 속성에 따라 변신 로봇처럼 형태가 바뀝니다.

|**타입**|**코드**|**용도**|
|---|---|---|
|**text**|`type="text"`|일반적인 글자 입력|
|**password**|`type="password"`|입력 시 글자가 마스킹(●●●) 처리됨|
|**checkbox**|`type="checkbox"`|여러 개 선택 가능 (체크박스)|
|**radio**|`type="radio"`|여러 개 중 딱 하나만 선택 (이름(`name`)이 같아야 함)|
|**file**|`type="file"`|내 컴퓨터 파일 업로드|
|**submit**|`type="submit"`|폼 내용을 전송하는 버튼|

## 4. Label 태그와 접근성 연결

`<label>`은 입력창에 이름을 붙여주는 태그인데, 그냥 쓰는 게 아니라 **`id`**와 연결해야 제맛입니다.

HTML

```
<label for="user-id">아이디</label>
<input type="text" id="user-id" name="userid">
```

- **연결 방법**: `label`의 `for` 값과 `input`의 `id` 값을 일치시킵니다.
    
- **장점**:
    
    1. **사용성**: "아이디"라는 글자만 클릭해도 입력창에 커서가 생깁니다. (손가락 두꺼운 분들에겐 축복!)
        
    2. **접근성**: 시각장애인이 스크린 리더를 쓸 때 "이 입력창은 아이디를 적는 곳이군요"라고 정확히 읽어줍니다.
       
- **전송 방식 (Method)**:
    
    - `GET`: 주소창에 데이터 노출 (검색 등).
        
    - `POST`: 봉투에 담아 숨겨 보냄 (로그인, 개인정보).
        
- **Input의 변신**: `email`, `tel`, `url` 등을 정확히 쓰면 모바일 키보드가 그에 맞게 바뀝니다 (`inputmode`).
    
- **Fieldset & Legend**: 관련 있는 입력창(예: 배송지 정보)을 테두리로 예쁘게 묶고 제목을 다는 법.
