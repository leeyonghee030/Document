## 1. A 태그와 절대경로/상대경로

`<a>` 태그는 'Anchor'의 약자로, 다른 페이지나 파일로 연결하는 통로입니다. 이때 주소를 적는 방식인 '경로'를 이해하는 것이 핵심입니다.

- **절대경로**: 전체 URL을 다 적는 방식입니다. 외부 사이트로 연결할 때 주로 사용합니다.
    
    - 예: `https://www.google.com`
        
- **상대경로**: 현재 내 파일의 위치를 기준으로 적는 방식입니다. 내 프로젝트 내부 파일을 연결할 때 사용합니다.
    
    - `./index.html` : 현재 폴더의 파일
        
    - `../images/logo.png` : 상위 폴더로 나가서 images 폴더 안의 파일
        
- **속성 팁**: `target="_blank"`를 넣으면 새 탭에서 링크가 열립니다.

## 2. IMG 태그와 Alt 속성의 중요성

이미지를 삽입할 때는 `<img>` 태그를 사용하며, 닫는 태그가 없습니다.

- **`src`**: 이미지 파일의 경로(Source)입니다.
    
- **`alt` (Alternative Text)**: **가장 중요한 속성**입니다. 이미지가 로딩되지 않았을 때 대신 보여줄 텍스트이며, 시각장애인이 사용하는 스크린 리더가 이 내용을 읽어줍니다.
    
- **SEO**: 검색 엔진은 이미지를 직접 볼 수 없기 때문에 `alt` 텍스트를 보고 이미지의 내용을 파악합니다.

## 3. Video 및 Audio 임베딩

HTML5부터는 별도의 플러그인 없이도 영상과 소리를 재생할 수 있습니다.

- **`<video>` 태그**: `controls` 속성을 넣어야 재생/일시정지 버튼이 나타납니다. `autoplay`, `loop`, `muted` 등의 속성으로 자동재생이나 반복을 설정할 수 있습니다.
    
- **`<audio>` 태그**: 비디오와 유사하게 `src`와 `controls`를 사용합니다.
  
  - **이미지 최적화**:
    
    - `alt=""`: 시각 장애인을 위한 설명 (의미 없으면 빈값이라도 꼭 넣기).
        
    - `loading="lazy"`: 화면에 보일 때 천천히 로딩해서 **페이지 속도 업!**
        
    - **반응형 이미지**: `srcset`(다양한 크기 제공)과 `picture`(포맷/상황별 이미지 교체) 활용.
        
- **비디오 & 오디오**:
    
    - `source`: 여러 포맷을 넣어 브라우저 호환성 해결.
        
    - `#t=시작,종료`: "1분 20초부터 재생해!" 같은 **시간 지정** 가능.
        
    - `track`: 자막(`subtitles`)이나 청각 장애인용 캡션(`captions`) 제공.
    
- **브라우저 호환성**: 여러 형식을 지원하기 위해 `<source>` 태그를 내부에 중첩해서 사용하는 것이 안전합니다.

## 4. Iframe 활용과 주의사항

`<iframe>`은 현재 페이지 안에 또 다른 웹 페이지를 액자처럼 끼워 넣는 태그입니다. 유튜브 영상 퍼가기나 구글 지도 삽입 시 가장 많이 쓰입니다.

- **활용**: 유튜브의 '공유 > 퍼가기' 코드가 바로 이 iframe입니다.
    
- **주의사항**:
    
    - **보안 문제**: 출처가 불분명한 사이트를 불러오면 보안 위험이 있습니다.
        
    - **성능**: 너무 많은 iframe은 웹 페이지 속도를 느리게 만듭니다.
        
    - **검색 엔진**: 검색 엔진 로봇은 iframe 내부의 콘텐츠를 잘 읽지 못하는 경우가 많습니다.
