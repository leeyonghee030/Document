> [!info] **학습 목표** 작성한 HTML 코드가 웹 표준을 준수하는지 확인하고, 브라우저별 호환성을 체크하며, 개발자 도구를 통해 실시간으로 코드를 수정하고 디버깅하는 능력을 키웁니다.

## 1. W3C Validator: 문법 오류 찾아주는 웹 표준 검사기

W3C(World Wide Web Consortium)는 웹의 표준을 만드는 기구입니다. 이곳에서 제공하는 검사기는 내 코드가 문법적으로 완벽한지 판별해 주는 '선생님' 같은 존재입니다.

- **사용 방법**: [W3C Validator 사이트](https://validator.w3.org/) 접속
    
    1. **Validate by URI**: 이미 배포된 사이트 주소 입력
        
    2. **Validate by File Upload**: 작성한 `.html` 파일 업로드
        
    3. **Validate by Direct Input**: 코드를 직접 복사해서 붙여넣기 (가장 많이 쓰임)
        
- **체크 포인트**:
    
    - **Error**: 반드시 고쳐야 할 문법 위반 (예: 태그 안 닫음, 중복 ID 사용 등)
        
    - **Warning**: 실행은 되지만 권장되지 않는 방식 (예: `section` 태그 안에 제목`<h1>~<h6>`이 없음)
        
- **왜 하나요?**: 문법이 깨끗해야 검색 엔진이 내 사이트를 잘 읽고, 브라우저가 화면을 그릴 때 예상치 못한 버그가 생기지 않습니다.

## 2. Can I Use: "이 기능, 다른 브라우저에서도 돌아갈까?"

프론트엔드 개발자의 최대 고민은 "내가 쓴 최신 기능이 사용자의 브라우저(크롬, 사파리, 엣지 등)에서도 잘 보일까?"입니다. 이를 한눈에 확인해 주는 사이트가 바로 [Can I Use](https://caniuse.com/)입니다.

- **활용 예시**:
    
    - 새로 나온 `<dialog>` 태그나 특정 CSS 속성을 써도 될지 궁금할 때 검색창에 입력합니다.
        
    - **초록색**: 지원 가능 / **빨간색**: 지원 불가 / **노란색**: 부분 지원.
        
- **실무 팁**:
    
    - 보통 글로벌 통계상 95% 이상의 지원율을 보이면 안심하고 사용합니다.
        
    - 만약 지원율이 낮다면 'Polyfill(폴리필)'이라 부르는 대체 코드를 준비해야 합니다.

## 3. Chrome DevTools: 실시간으로 내 코드 뜯어보기

브라우저에는 개발자를 위한 강력한 도구가 내장되어 있습니다. 화면을 보면서 즉석에서 코드를 수정해 볼 수 있는 '실험실'입니다.

- **실행 방법**: 키보드 `F12` 또는 `Ctrl + Shift + I` (맥은 `Cmd + Opt + I`)
    
- **주요 기능 (Elements 탭)**:
    
    - **Inspect(검사)**: 도구 왼쪽 상단의 화살표 아이콘을 누르고 화면의 특정 요소를 클릭하면, 해당 HTML 코드로 즉시 이동합니다.
        
    - **Live Edit**: 개발자 도구 창에서 HTML 태그를 더블 클릭해 내용을 바꾸거나 삭제할 수 있습니다. (실제 파일이 바뀌는 건 아니니 안심하고 테스트하세요!)
        
    - **Console**: 자바스크립트 에러 메시지를 확인하거나 직접 코드를 입력해 결과를 확인합니다.
