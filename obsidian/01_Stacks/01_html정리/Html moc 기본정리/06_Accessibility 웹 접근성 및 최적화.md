

# [HTML] 06_Accessibility: 웹 접근성 및 최적화

> [!info] **학습 목표** 장애가 있거나 고령인 사용자를 포함한 모든 사람이 웹사이트를 동등하게 이용할 수 있도록 하는 **웹 접근성(A11y)**과, 페이지를 더 빠르게 로드하는 **최적화 방법**을 배웁니다.

---

## 1. 웹 접근성 기초 (A11y)

'A11y'는 Accessibility의 첫 글자 'A'와 마지막 글자 'y' 사이의 글자 수가 11개라는 뜻에서 유래했습니다.

- **접근성이 필요한 이유**:
    
    - **포용성**: 시각장애인(스크린 리더 사용), 청각장애인, 혹은 마우스 사용이 불편한 사람들도 정보를 얻을 수 있어야 합니다.
        
    - **법적 준수**: 많은 국가에서 웹 접근성 준수를 법으로 규정하고 있습니다.
        
    - **SEO 향상**: 접근성이 좋은 사이트는 검색 엔진이 정보를 파악하기 쉬워 검색 순위가 올라갑니다.
        

---

## 2. ARIA Roles (WAI-ARIA)

WAI-ARIA는 HTML의 부족한 의미를 보충해 주는 도구입니다. 하지만 가장 중요한 원칙은 **"가능하면 시맨틱 태그를 쓰고, ARIA는 최후의 수단으로 쓴다"**입니다.

- **`role` 속성**: 요소의 역할을 명시합니다.
    
    - 예: `<div role="button">클릭</div>` (스크린 리더가 이 div를 버튼으로 인식하게 함)
        
- **`aria-label`**: 화면에 텍스트가 없어도 스크린 리더가 읽어줄 이름을 지정합니다.
    
    - 예: `<button aria-label="닫기">X</button>`
        
- **`aria-hidden="true"`**: 디자인 요소일 뿐 사용자가 알 필요 없는 내용은 스크린 리더가 건너뛰게 합니다.
    

---

## 3. HTML 파일 최적화 팁 (Optimization)

사용자가 사이트에 들어왔을 때 '빠르다'고 느끼게 만드는 기술적인 습관들입니다.

- **DOM 구조 단순화**: 태그를 너무 깊게 중첩(Deep nesting)하지 마세요. 브라우저가 화면을 그리는 속도가 느려집니다.
    
- **이미지 최적화**:
    
    - 적절한 크기의 이미지를 사용하세요.
        
    - `loading="lazy"` 속성을 사용하여 화면에 보일 때만 이미지를 불러오도록 설정하세요.
        
- **스크립트 위치**: 자바스크립트(`<script>`) 파일은 가급적 `</body>` 태그 바로 직전에 넣거나, `defer` 속성을 사용해 HTML 해석이 방해받지 않게 하세요.
    
    - 예: `<script src="app.js" defer></script>`
        
- **불필요한 코드 제거**: 사용하지 않는 주석이나 공백을 줄여 파일 용량을 최소화합니다.
