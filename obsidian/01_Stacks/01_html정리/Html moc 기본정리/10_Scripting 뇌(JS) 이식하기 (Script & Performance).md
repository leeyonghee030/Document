## 1. Script 태그: 외부 JS 파일 연결법

자바스크립트는 웹사이트의 '동작'을 담당합니다. HTML 파일 안에 직접 적을 수도 있지만, 관리의 편의성을 위해 외부 파일(`.js`)로 분리하여 연결하는 것이 정석입니다.

- **기본 문법**:
    
    HTML
    
    ```
    <script src="js/script.js"></script>
    ```
    
- **연결 위치의 역사**:
    
    - **과거**: 보통 `</body>` 태그 바로 직전에 넣었습니다. HTML을 다 읽기 전에 무거운 JS를 읽으면 화면이 늦게 뜨기 때문입니다.
        
    - **현재**: 후술할 `defer` 속성 덕분에 다시 `<head>` 안으로 옮겨가는 추세입니다.

## 2. Async vs Defer: 로딩 성능의 마법사

브라우저는 기본적으로 HTML을 읽다가 `<script>`를 만나면 읽기를 멈추고 JS를 다운로드/실행합니다. 이 '멈춤' 현상을 해결하기 위해 두 가지 마법의 단어가 존재합니다.

### ① Default (속성 없음)

- **동작**: HTML 파싱 중단 → JS 다운로드 및 실행 → 다시 HTML 파싱.
    
- **문제점**: JS 파일이 크면 사용자는 빈 화면을 오래 보게 됩니다.
    

### ② Async (비동기)

- **동작**: "다운로드는 백그라운드에서 할게, 다 되면 바로 실행할게!"
    
- **특징**: 다운로드가 끝나자마자 즉시 실행됩니다. 이때 HTML 파싱이 잠시 중단됩니다.
    
- **단점**: 실행 순서가 보장되지 않습니다. (먼저 다운로드된 놈이 먼저 실행됨)
    

### ③ Defer (지연) - **가장 추천!**

- **동작**: "다운로드는 백그라운드에서 하고, HTML 다 읽은 다음에 실행할게!"
    
- **특징**: HTML 파싱을 방해하지 않으며, 페이지가 완전히 준비된 후 스크립트가 순서대로 실행됩니다.
    
- **장점**: 실행 순서가 보장되며 성능상 가장 유리합니다.
    

|**속성**|**HTML 파싱 중단 여부**|**실행 시점**|**실행 순서 보장**|
|---|---|---|---|
|**없음**|**YES** (다운로드+실행 시)|즉시|YES|
|**async**|**YES** (실행 시에만)|다운로드 직후|NO|
|**defer**|**NO**|HTML 파싱 완료 후|YES|

## 3. NoScript: 자바스크립트가 꺼진 환경을 위한 배려

매우 드문 경우지만, 보안상의 이유로 자바스크립트를 비활성화한 사용자나 오래된 브라우저 환경을 위해 사용합니다.

- **역할**: 자바스크립트를 지원하지 않는 환경에서만 화면에 나타나는 문구를 지정합니다.
    
- **코드 예시**:
    
    HTML
    
    ```
    <noscript>
      <div style="color: red;">
        이 사이트는 자바스크립트가 활성화되어야 정상적으로 작동합니다. 
        브라우저 설정에서 자바스크립트를 켜주세요!
      </div>
    </noscript>
    ```

## 💡 프론트엔드 개발자의 성능 최적화 팁

1. **묻지도 따지지도 말고 `defer`**: 일반적인 라이브러리나 메인 스크립트는 `<head>` 영역에 넣고 `defer` 속성을 부여하는 것이 현재 표준(Best Practice)입니다.
    
2. **`async`의 용도**: 구글 애널리틱스나 광고 스크립트처럼 독립적으로 돌아가도 되고, 페이지 기능에 직접적인 영향을 주지 않는 외부 서비스 연결 시에 주로 사용합니다.
    
3. **인라인 스크립트 최소화**: HTML 안에 `<script> console.log('hi'); </script>` 처럼 직접 코드를 짜는 것은 보안(XSS 공격)과 캐싱 측면에서 좋지 않습니다. 가급적 외부 파일로 빼세요.
