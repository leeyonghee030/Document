> [!info] **학습 목표** HTML 문법에서 특별한 의미를 갖는 문자(예: `<`, `>`)를 텍스트로 안전하게 표시하는 방법과, 키보드에 없는 특수 기호 및 이모지를 웹 표준에 맞게 사용하는 법을 익힙니다.

---

## 1. 예약어 탈출하기 (`<`, `>`, `&`)

HTML에서 `<`와 `>`는 태그를 여닫는 용도로 미리 약속되어 있습니다. 만약 본문에 "1 < 2"라고 그냥 쓰면 브라우저는 `< 2`를 새로운 태그의 시작으로 오해해 레이아웃을 깨뜨릴 수 있습니다.

- **`&lt;` (Less Than)**: `<` 기호를 표시합니다.
    
- **`&gt;` (Greater Than)**: `>` 기호를 표시합니다.
    
- **`&amp;` (Ampersand)**: `&` 기호 자체를 표시합니다. (엔티티 코드의 시작점이라 본인도 예약어입니다!)
    
- **예시**:
    
    HTML
    
    ```
    <code>if (a &lt; b &amp;&amp; b &gt; c)</code>
    ```
    

---

## 2. 공백 제어하기 (`&nbsp;`)

HTML은 소스 코드상에서 아무리 스페이스바를 많이 눌러도 화면에는 **단 한 칸**의 공백만 보여줍니다. 여러 칸을 띄우거나 줄바꿈을 막고 싶을 때 엔티티를 사용합니다.

- **`&nbsp;` (Non-Breaking Space)**: 줄바꿈 없는 공백입니다.
    
    - 단순히 여러 칸을 띄울 때도 쓰지만, 단어 사이가 벌어져서 줄바꿈 되는 것을 방지하는 ‘접착제’ 역할도 합니다.
        
- **예시**:
    
    HTML
    
    ```
    <p>세&nbsp;&nbsp;&nbsp;칸 띄우기</p>
    <p>절대_떨어지면_안_되는_단어&nbsp;결합</p>
    ```
    

---

## 3. 자주 쓰는 기호 (Copyright 등)

키보드에 없거나 입력하기 번거로운 기호들도 엔티티 코드로 간단히 입력 가능합니다.

- **`&copy;`**: 저작권 표시 (©)
    
- **`&reg;`**: 등록 상표 표시 (®)
    
- **`&deg;`**: 각도 단위 (°)
    
- **`&plusmn;`**: 플러스마이너스 (±)
    
- **예시**:
    
    HTML
    
    ```
    <footer>
      <p>&copy; 2026 My Coding Blog. All rights reserved.</p>
    </footer>
    ```

## 4. Emojis: 이모지를 안전하게 쓰는 법 🚀

현대 웹에서는 이모지를 직접 입력해도 대부분 잘 나오지만(UTF-8 설정 덕분), 가끔 환경에 따라 깨질 우려가 있거나 좀 더 전문적인 방식을 원한다면 **Unicode(십진수/십육진수)** 코드를 사용합니다.

- **방법 1: 직접 입력 (가장 흔함)**
    
    - 맥: `Cmd + Ctrl + Space` / 윈도우: `Win + .`
        
- **방법 2: 십진수/십육진수 코드 (가장 안전)**
    
    - `&#128640;` → 🚀
        
    - `&#128218;` → 📚
        
- **웹 접근성 팁**: 이모지는 스크린 리더가 읽기 어려울 수 있습니다. 의미가 중요하다면 `role="img"`와 `aria-label`을 함께 써주는 것이 좋습니다.
    
    HTML
    
    ```
    <span role="img" aria-label="rocket">🚀</span>
    ```

## 💡 프론트엔드 개발자의 실무 팁

1. **엔티티 코드 외우지 마세요**: `&lt;`, `&gt;`, `&nbsp;`, `&amp;` 정도만 기억하고 나머지는 필요할 때 **[HTML Entities Cheat Sheet]**를 검색해서 복사해 쓰면 됩니다.
    
2. **레이아웃용   자제**: 여러 칸을 띄우기 위해 `&nbsp;&nbsp;&nbsp;`를 연달아 쓰는 것은 좋지 않은 습관입니다. 간격 조절은 CSS의 `margin`이나 `padding`으로 해결하는 것이 정석입니다.
    
3. **UTF-8 인코딩 확인**: `<meta charset="UTF-8">`이 설정되어 있다면 대부분의 특수문자와 이모지는 직접 입력해도 잘 작동합니다. 하지만 코드 안에서 기호 자체가 문법적 오류를 일으킬 것 같을 때만 엔티티를 쓰세요.
