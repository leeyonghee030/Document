# 📑 JavaScript 핵심 요약: 메서드와 this


---

## 🟢 1. 메서드(Method)란?
객체 프로퍼티에 할당된 함수를 **메서드**라고 부릅니다. 객체가 "인사하기", "데이터 처리하기"와 같은 특정 행동을 할 수 있게 해주는 기능입니다.

### ✅ 메서드 선언 방식
```javascript
let user = {
  name: "John",
  
  // 1️⃣ 방법 1: 프로퍼티에 함수를 직접 할당
  sayHi: function() {
    console.log("Hello");
  },

  // 2️⃣ 방법 2: 메서드 단축 구문 (🌟 실무 권장: 더 깔끔함!)
  sayBye() {
    console.log("Bye");
  }
};
````

---

## 🔵 2. 메서드와 `this`

메서드는 자기가 속한 객체의 정보를 활용해야 할 때가 많습니다. 이때 `this`를 사용하면 객체 내부 데이터에 접근할 수 있습니다.

- **`this`의 정체**: 메서드를 호출할 때 사용된 **'점(.) 앞'의 객체**입니다.
    
- **사용 이유**: 객체 이름(`user.name`)을 직접 쓰면 객체가 복사되거나 변수명이 바뀌었을 때 코드가 깨질 수 있습니다. 반면 `this`는 호출 시점의 대상을 자동으로 가리키므로 안전합니다.
    

---

## 🟠 3. 자바스크립트 `this`의 특징: 자유로움!

자바스크립트의 `this`는 함수가 선언된 곳이 아니라, **"함수가 호출된 시점"**에 결정됩니다. 이를 **'런타임에 결정된다'**고 합니다.

JavaScript

```
let user = { name: "John" };
let admin = { name: "Admin" };

function sayHi() {
  console.log(this.name); // 💡 호출되기 전까지 this는 정해지지 않음!
}

// 두 객체에 동일한 함수를 나눠줌
user.f = sayHi;
admin.f = sayHi;

// 호출하는 주체(점 앞의 객체)에 따라 this가 변함
user.f();  // ➔ John (this == user)
admin.f(); // ➔ Admin (this == admin)
```

> **⚠️ 💡 참고**: 객체 없이 단독으로 호출(`sayHi()`)하면 엄격 모드(`'use strict'`)에서 `this`는 `undefined`가 됩니다.

---

## 🔴 4. this가 없는 화살표 함수 🏹

화살표 함수(`=>`)는 일반 함수와 달리 자신만의 고유한 `this`를 가지지 않습니다.

- **작동 방식**: 화살표 함수 내에서 `this`를 쓰면, 화살표 함수를 감싸고 있는 **외부 일반 함수**의 `this`를 그대로 가져옵니다. (이를 **'렉시컬 this'**라고 부릅니다!)
    
- **언제 쓰나요?**: 메서드 내부에서 별도의 콜백 함수를 쓸 때, 외부의 `this`를 그대로 유지하고 싶을 때 매우 유용합니다.
    

---

## ✨ 💡 5. 핵심 요약 (Cheat Sheet)

|**개념**|**한 줄 정리**|**🔥 핵심 포인트**|
|---|---|---|
|**메서드**|객체에 담긴 함수|객체의 '행동'을 정의함|
|**this**|**"나를 부른 점(.) 왼쪽의 주인님"**|호출 주체에 따라 **동적**으로 변함|
|**자유로운 this**|어떤 객체에서도 재사용 가능|선언 시점이 아닌 **호출 시점**이 중요|
|**화살표 함수**|고유한 `this`가 없음|부모 함수의 `this`를 빌려다 씀|

# 🚀 JavaScript 실습: 객체 메서드와 체이닝

---

## 🧮 1. 계산기 객체 (this를 통한 상태 저장)
`this`를 사용하여 사용자로부터 입력받은 값을 객체의 프로퍼티(`this.a`, `this.b`)에 저장하고 이를 활용하는 예제입니다.

### ✅ 구현 코드
```javascript
let calculator = {
  // 1️⃣ 값 읽기: 입력받은 값을 숫자로 변환하여 객체 프로퍼티에 저장
  read() { 
    this.a = +prompt("첫 번째 값", 0);
    this.b = +prompt("두 번째 값", 0);
  },
  
  // 2️⃣ 더하기: 저장된 프로퍼티 사용
  sum() {
    return this.a + this.b;
  },
  
  // 3️⃣ 곱하기: 저장된 프로퍼티 사용
  mul() {
    return this.a * this.b;
  }
};

calculator.read();
alert( "합: " + calculator.sum() );
alert( "곱: " + calculator.mul() );

---

## 🪜 2. 사다리 객체 (메서드 체이닝)

메서드가 수행을 마친 후 **`return this`**를 통해 객체 자신을 반환하면, 점(`.`)을 찍어 연쇄적으로 메서드를 호출할 수 있습니다.

### ✅ 구현 코드
```
let ladder = {
  step: 0,

  up() {
    this.step++;
    return this; // 🌟 핵심: 객체 자신을 반환!
  },

  down() {
    this.step--;
    return this; // 🌟 핵심: 객체 자신을 반환!
  },

  showStep() {
    alert( this.step );
    return this; // 🌟 차후 연결을 위해 반환
  }
};

// 🔥 메서드 체이닝 실행
ladder.up().up().down().up().down().showStep(); // 결과: 1

# 📦 JS 기초: 생성자 함수와 `new` 연산자 완벽 정리

객체를 하나하나 만드는 것이 '수제 쿠키'를 굽는 것이라면, **생성자 함수**는 똑같은 모양의 쿠키를 계속 찍어낼 수 있는 **'쿠키 틀'**을 만드는 것과 같습니다.

---

## 1. 생성자 함수란? (The Constructor)
유사한 객체를 여러 개 만들어야 할 때 사용하는 "객체 제작용 함수"입니다.

### ✅ 두 가지 약속
1. **이름의 첫 글자는 대문자**로 시작합니다 (예: `User`, `Item`).
2. 반드시 **`new` 연산자**를 붙여서 실행합니다.



```javascript
function User(name) {
  // 1. (암시적) this = {};  빈 객체가 만들어짐
  
  this.name = name;
  this.isAdmin = false;
  
  // 2. (암시적) return this; 객체가 반환됨
}

let user1 = new User("철수");
let user2 = new User("영희");
````

---

## 2. `new`를 붙이면 내부에서 일어나는 일

`new` 연산자와 함께 함수를 호출하면 자바스크립트 엔진은 보이지 않는 곳에서 다음과 같이 동작합니다.

1. **빈 객체 생성**: `this = {}`가 만들어집니다.
    
2. **속성 추가**: `this`에 프로퍼티와 메서드들을 채워 넣습니다.
    
3. **자동 반환**: 함수 끝에 `return this`가 없어도 완성된 객체를 자동으로 내뱉습니다.
    

---

## 3. `new.target` (보너스 체크)

"이 함수가 `new`와 함께 호출되었나?"를 확인하는 도구입니다.

- `new`와 함께라면: **함수 자신**을 반환
    
- 일반 호출이라면: **undefined**를 반환
    

> 💡 **Tip:** `new`를 생략해도 자동으로 붙여주는 코드를 짤 수는 있지만, 가독성을 위해 항상 `new`를 직접 쓰는 것이 좋습니다.

---

## 4. 생성자에서의 `return`문

보통 생성자 함수는 `return`을 쓰지 않습니다. 하지만 굳이 쓴다면 아래 규칙을 따릅니다.

|**return 하는 값**|**결과**|
|---|---|
|**객체 (`{...}`)**|`this`를 무시하고 **그 객체**를 반환 (강제 교체)|
|**원시값 (숫자, 문자 등)**|return문을 **무시**하고 원래대로 `this` 반환|

---

## 5. 메서드 추가하기

생성자 함수 내부에서 함수를 정의하면, 만들어지는 모든 객체가 해당 메서드를 가질 수 있습니다.

JavaScript

```
function Counter() {
  this.value = 0;

  this.increase = function() {
    this.value++;
  };
}

let myCounter = new Counter();
myCounter.increase();
console.log(myCounter.value); // 1
```

---

## 💡 핵심 요약 (Cheat Sheet)

- **생성자 함수**는 객체를 찍어내는 틀이다.
    
- **`new`**를 붙여야만 객체가 생성되고 `this`가 반환된다.
    
- 함수 이름은 **대문자**로 시작해서 "나 생성자야!"라고 티를 낸다.
    
- 객체 내부에 동작을 넣고 싶다면 `this.메서드명 = function() {...}`을 사용한다.


# 📑 JS 학습 노트: 생성자 함수와 `new` 연산자

## 1. 생성자 함수에서 객체 반환하기 (A와 B의 비밀)

일반적으로 생성자 함수는 `this`에 새로운 객체를 만들어 반환하지만, **명시적으로 `return` 문에 객체를 적어주면** 그 객체가 반환됩니다.

JavaScript

```
let obj = {};

function A() { return obj; } // 새로운 this 대신 외부의 obj를 반환
function B() { return obj; } // 마찬가지로 외부의 obj를 반환

alert( new A() == new B() ); // true
```

- **해설**: `new A()`와 `new B()`가 서로 다른 이름을 가진 생성자임에도 불구하고, 결과적으로는 둘 다 **똑같은 외부 객체 `obj`**의 주소값을 참조하게 됩니다. 그래서 비교 연산 시 `true`가 나옵니다.
    

---

## 2. 계산기 생성자 함수 (Calculator)

동일한 구조의 계산기 객체를 여러 개 만들 수 있도록 틀(Template)을 만드는 과정입니다.

JavaScript

```
function Calculator() {
  // 1. 값 입력 메서드
  this.read = function() {
    this.a = +prompt('a?', 0);
    this.b = +prompt('b?', 0);
  };

  // 2. 더하기 메서드
  this.sum = function() {
    return this.a + this.b;
  };

  // 3. 곱하기 메서드
  this.mul = function() {
    return this.a * this.b;
  };
}

let calculator = new Calculator();
calculator.read();
alert("Sum=" + calculator.sum());
alert("Mul=" + calculator.mul());
```

- **해설**: `new Calculator()`를 호출하면 빈 객체 `this`가 만들어지고, 그 안에 `read`, `sum`, `mul` 함수들이 담겨서 나옵니다.
    

---

## 3. 누산기 생성자 함수 (Accumulator)

객체에 값을 계속해서 누적(Accumulate)시키는 예제입니다.

JavaScript

```
function Accumulator(startingValue) {
  // 처음 시작 값을 객체의 value에 저장
  this.value = startingValue;

  // 호출할 때마다 prompt에서 받은 값을 value에 더함
  this.read = function() {
    this.value += +prompt('더할 값을 입력해주세요.', 0);
  };
}

let accumulator = new Accumulator(1); // 초기값 1로 시작
accumulator.read(); // 사용자가 5를 입력하면 1 + 5 = 6
accumulator.read(); // 사용자가 10을 입력하면 6 + 10 = 16
alert(accumulator.value); // 최종값 출력
```

- **해설**: 생성자 함수가 매개변수(`startingValue`)를 받을 수 있다는 점이 핵심입니다. 이를 통해 각기 다른 초기값을 가진 여러 개의 누산기를 만들 수 있습니다.
