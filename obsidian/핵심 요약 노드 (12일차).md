# 📚 자바스크립트 배열(Array) 완전 정복

## 1. 배열의 기초: 왜 객체 대신 배열인가?

객체는 '키(Key)' 중심이라 **순서**를 보장하지 않습니다. 반면, 배열은 **인덱스(0, 1, 2...)**를 사용하여 데이터의 순서를 완벽하게 관리합니다.

- **선언**: `let arr = ["사과", "바나나"];` (대괄호 권장)
    
- **특징**: 요소의 자료형에 제한이 없음 (숫자, 객체, 함수 등 혼합 가능)
    
- **length**: 배열의 요소 개수가 아니라, **'가장 큰 인덱스 + 1'**을 의미합니다.


---

## 2. 배열의 핵심 메서드 (데큐, Deque)

자바스크립트 배열은 앞뒤 양쪽에서 요소를 넣고 뺄 수 있는 **데큐(Double Ended Queue)**처럼 동작합니다.

### 🚀 끝에 작업하기 (매우 빠름)

- **`push(...items)`**: 맨 끝에 요소 추가
    
- **`pop()`**: 맨 끝 요소 제거 및 반환
    

### 🐢 앞에 작업하기 (상대적으로 느림)

- **`unshift(...items)`**: 맨 앞에 요소 추가
    
- **`shift()`**: 맨 앞 요소 제거 및 반환
    

> **왜 앞쪽 작업(shift/unshift)이 느린가요?** 배열의 첫 번째 요소를 건드리면, 뒤에 있는 **모든 요소의 인덱스를 하나씩 당기거나 밀어야** 하기 때문입니다. 반면 끝쪽 작업은 다른 요소를 건드릴 필요가 없습니다.

## 4. 배열의 내부 동작과 최적화

배열은 본질적으로 **객체**입니다. 하지만 자바스크립트 엔진은 배열을 일반 객체와 다르게 **연속된 메모리 공간**에 배치하여 최적화합니다.

### ⚠️ 최적화가 깨지는 나쁜 습관

- `arr[99999] = 5;` 처럼 인덱스를 건너뛰는 경우 (Sparse Array)
    
- `arr.age = 25;` 처럼 숫자가 아닌 프로퍼티를 추가하는 경우
    
- 요소를 역순으로 채우는 경우
    

---

## 5. 독특한 특징들 (주의사항)

### 📏 `length` 프로퍼티의 마법

`length`는 읽기뿐만 아니라 **쓰기도 가능**합니다.

- `arr.length = 0;`: 배열을 즉시 비우는 가장 빠른 방법입니다.
    
- `arr.length = 2;`: 배열을 앞에서부터 2개만 남기고 **잘라버립니다.** (복구 불가능)
    

### 🧩 `new Array()`의 함정

- `new Array(5)`라고 하면 `[5]`가 아니라, **비어있는 5개짜리 공간**이 만들어집니다. 결과는 `[undefined, undefined, ...]`와 같아 실수를 유발하기 쉽습니다.
    

### 🧵 `toString` 형 변환

배열이 연산 중에 문자열로 변환되면 요소를 **쉼표(,)로 구분한 문자열**이 됩니다.

- `[1, 2] + 1` → `"1,2" + 1` → **`"1,21"`**
    
- `[] + 1` → `"" + 1` → **`"1"`**

## 6. 자료구조로서의 활용

### 🏗️ 스택 (Stack) - LIFO (후입선출)

- **사용 메서드**: `push` / `pop`
    
- **비유**: 쌓여있는 접시 (가장 마지막에 놓은 접시를 가장 먼저 집음)
    

### 큐 (Queue) - FIFO (선입선출)

- **사용 메서드**: `push` / `shift`
- **비유**: 편의점의 선입선출 진열 (먼저 들어온 우유가 먼저 나감)



## 1. 요소를 추가하거나 삭제하기 (수정 & 변형)

배열의 특정 위치에 데이터를 넣거나 빼는 방법입니다.

### ✂️ 만능 도구: `splice`

가장 강력한 메서드입니다. **요소 삭제, 추가, 교체**를 한 번에 할 수 있습니다.

- **문법**: `arr.splice(시작인덱스, 삭제할개수, 추가할요소1, ...)`
    
- **특징**: 원본 배열이 직접 변합니다. 삭제된 요소들을 배열로 반환합니다.
    

### 🧊 안전한 복사: `slice`

원본은 건드리지 않고 **일부분만 복사**해서 가져옵니다.

- **문법**: `arr.slice(시작, 끝)` (끝 인덱스는 포함되지 않음)
    
- **꿀팁**: `arr.slice()`처럼 인자 없이 쓰면 전체 복사본을 만듭니다.
    

### 🔗 합치기: `concat`

여러 배열이나 값을 하나로 합쳐서 **새로운 배열**을 만듭니다.

---

## 2. 배열에서 원하는 값 찾기

### 🔍 위치와 존재 확인

- **`indexOf(값)`**: 앞에서부터 찾아 인덱스 반환 (없으면 `-1`).
    
- **`includes(값)`**: 배열에 그 값이 있는지 `true/false`로 확인. (**NaN도 찾을 수 있어 편리함!**)
    

### 🎯 조건으로 찾기 (`find`, `filter`)

배열 안에 객체가 들어있을 때 특히 유용합니다.

- **`find(함수)`**: 조건을 만족하는 **첫 번째 요소 하나**만 반환.
    
- **`filter(함수)`**: 조건을 만족하는 **모든 요소를 모아 새로운 배열**로 반환.
    

---

## 3. 배열 전체를 반복하며 가공하기 (핵심!)

### 🔄 단순히 반복만: `forEach`

모든 요소를 한 번씩 돌면서 특정 작업을 수행합니다. (반환값 없음)

### 🧪 새로운 배열로 변신: `map`

가장 많이 쓰이는 메서드 중 하나입니다. 각 요소를 함수로 가공해서 **새로운 배열**을 만듭니다.

- **예시**: 숫자가 든 배열을 모두 2배로 만든 배열로 바꿀 때 사용.
    

---

## 4. 배열 정렬하고 뒤집기

- **`sort(비교함수)`**: 배열을 정렬합니다.
    
    - **주의**: 기본적으로 요소를 **문자열**로 취급합니다. 숫자 정렬 시에는 반드시 `(a, b) => a - b` 같은 비교 함수를 넣어줘야 합니다.
        
- **`reverse()`**: 배열의 순서를 거꾸로 뒤집습니다.
    

---

## 5. 값 하나로 응축하기: `reduce`

배열의 모든 요소를 돌면서 하나의 최종 결과값(숫자 합계, 문자열 결합 등)을 도출합니다.

- **구성**: `arr.reduce((누산기, 현재값) => { ... }, 초깃값)`
    
- **초깃값 설정**: 빈 배열에서 에러가 나는 것을 방지하기 위해 **초깃값(보통 0이나 '')을 꼭 넣어주는 것이 안전**합니다.

## ⚠️ 초보자가 자주 하는 실수

1. **`sort()`의 함정**: `[1, 2, 15].sort()`를 하면 `[1, 15, 2]`가 됩니다. (사전순 정렬이기 때문) 숫자는 꼭 정렬 함수를 쓰세요!
    
2. **`undefined` 주의**: `find`나 `indexOf`로 값을 못 찾으면 각각 `undefined`와 `-1`을 뱉습니다. 이 결과를 체크하는 코드를 잊지 마세요.
    
3. **원본 훼손**: `splice`, `sort`, `reverse`는 원본을 바꿉니다. 원본을 유지해야 한다면 먼저 `slice()`로 복사한 뒤 사용하세요.
