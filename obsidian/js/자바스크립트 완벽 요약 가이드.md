# 🚀 자바스크립트 완벽 요약 가이드: 첫걸음부터 실전까지

## 1. 자바스크립트 첫걸음과 '변수'

자바스크립트에게 일을 시키기 위한 가장 기초적인 방법들을 배웁니다.

- **엄격 모드 (`"use strict";`)**: 코드를 더 꼼꼼하고 엄격하게 검사해달라고 컴퓨터에게 부탁하는 마법의 주문입니다.
    
- **출력하기 (`console.log`)**: 컴퓨터(콘솔창)에 원하는 글자나 결과를 띄워볼 수 있습니다.
    
- **변수 (`let`)**: 데이터를 담아두는 **'이름표가 붙은 상자'**입니다. 내용물을 언제든 바꿀 수 있습니다. (과거에는 `var`를 썼지만 요즘은 쓰지 않아요!)
    
- **상수 (`const`)**: 절대로 내용물을 바꿀 수 없는 **'자물쇠가 채워진 상자'**입니다 (예: 생일).
    
- **이름 짓기 규칙**: `a`, `b` 같은 대충 지은 이름보다는 `userName`처럼 누가 봐도 알기 쉬운 이름을 지어야 합니다.

## 2. 데이터의 종류와 브라우저 팝업창

컴퓨터가 인식하는 데이터의 종류를 알고, 사용자와 대화하는 법을 배웁니다.

- **데이터의 종류 (자료형)**: 숫자, 문자열(따옴표나 백틱으로 감쌈), 불리언(참 `true` / 거짓 `false`), `null`(일부러 비워둠), `undefined`(아직 값이 없음) 등이 있습니다.
    
- **사용자와 대화하는 3가지 팝업창**:
    
    - `alert()`: "알려줍니다!" (확인 버튼만 있는 경고창)
        
    - `prompt()`: "물어봅니다!" (글자를 입력받는 창)
        
    - `confirm()`: "확인받습니다!" (확인/취소 버튼으로 `true`/`false`를 얻음)
        
- **형 변환**: 문자를 숫자로 바꾸려면 `Number(변수)`나 앞에 `+` 기호를 붙여줍니다. 숫자가 아닌 것을 계산하려 하면 `NaN`(Not a Number)이라는 에러 값이 나옵니다. `isNaN()` 함수로 숫자가 맞는지 미리 검사할 수 있습니다.

## 3. 프로그램의 흐름 제어 (계산, 조건문, 반복문)

수학 계산을 하고, 상황에 따라 다른 행동을 하거나 반복하도록 지시합니다.

- **수학 기호**: 더하기(`+`), 빼기(`-`), 곱하기(`*`), 나누기(`/`) 외에, 나머지 값을 구하는 `%`와 거듭제곱을 구하는 `**`가 있습니다.
    
- **비교하기 (`===`)**: 두 개가 완벽히 똑같은지 비교할 때는 `==`보다, 데이터 종류까지 꼼꼼히 따지는 `===`(일치 연산자)를 쓰는 것이 훨씬 안전합니다.
    
- **논리 연산자**:
    
    - `&&` (AND): "이것도 맞고 **그리고** 저것도 맞아야 해!"
        
    - `||` (OR): "이거 **또는** 저거 둘 중 하나만 맞아도 통과!"
        
    - `!` (NOT): "반대로 뒤집어!" (참은 거짓으로, 거짓은 참으로)
        
- **조건문 (`if` / `else`)**: "만약 조건이 맞으면 이걸 하고(`if`), 아니면 저걸 해(`else`)!"라고 지시합니다.
    
    - **물음표 연산자 (`?`)**: `if`문을 아주 짧게 줄여 쓰는 방법입니다. (`조건 ? 맞을때 : 틀릴때` 형태).
        
- **반복문 (`for` / `while`)**:
    
    - `for`: "1번부터 10번까지 정확히 10번 반복해!"처럼 횟수가 정해져 있을 때 씁니다.
        
    - `while`: "숫자 6이 나올 때까지 계속 주사위 던져!"처럼 언제 끝날지 모를 때 조건을 걸고 씁니다.
        
- **멈추기와 건너뛰기**: `break`는 반복을 아예 끝내버리고, `continue`는 이번 턴만 건너뛰고 다음 반복으로 넘어갑니다.

## 4. 코드 재사용하기 (스위치와 함수)

코드를 깔끔하게 정리하고, 언제든 다시 불러 쓸 수 있는 덩어리로 만듭니다.

- **스위치 (`switch`)**: `if`문이 너무 길어질 때 씁니다. 값에 따라 스위치를 켜듯 `case`를 찾아가며, 볼일이 끝나면 `break`로 빠져나옵니다.
    
- **함수 (`function`)**: 여러 줄의 코드를 하나로 묶어 이름을 붙여둔 **'명령어 세트'**입니다. 보통 `show`(보여줘), `calc`(계산해) 같은 동사로 이름을 짓습니다.
    
- **결과 반환 (`return`)**: 함수 안에서 계산한 최종 결과를 함수 밖으로 뱉어낼 때 사용합니다.
    
- **함수를 만드는 3가지 방법**:
    
    1. 함수 선언문: 가장 기본 형태 (`function 이름() { ... }`)
        
    2. 함수 표현식: 변수 안에 함수를 집어넣기 (`let 이름 = function() { ... }`)
        
    3. 화살표 함수: 가장 짧고 세련된 최신 방식 (`let 이름 = () => { ... }`)

## 5. 객체(Object)의 비밀과 메모리

관련된 데이터와 함수들을 하나의 꾸러미로 묶는 방법을 배웁니다.

- **객체 (Object)**: 여러 정보(이름, 나이 등)와 행동(메서드)을 하나의 덩어리로 묶은 바구니입니다.
    
- **참조(Reference) 복사**: 변수에 객체를 저장하면 바구니 전체가 들어가는 게 아니라, 바구니가 있는 **'메모리 주소'**만 저장됩니다.
    
    - **주의점**: `let user2 = user;` 처럼 복사하면 두 변수가 **같은 바구니**를 가리키게 되어, 하나를 수정하면 다른 하나도 같이 변합니다.
        
- **메서드(Method)와 `this`**:
    
    - 객체 안에 들어있는 함수를 '메서드'라 부르며, 객체의 '행동'을 담당합니다.
        
    - **`this`**: 함수가 실행될 때, 그 함수를 부른 객체(점 `.` 바로 앞의 이름)를 가리킵니다. 누가 부르느냐에 따라 `this`는 계속 변합니다.
        
    - **화살표 함수와 `this`**: 화살표 함수(`=>`)는 자기만의 `this`가 없어서, 자기를 감싸고 있는 바깥쪽의 `this`를 그대로 빌려다 씁니다.
        
- **생성자 함수 (`new`)**: 똑같이 생긴 객체를 무한대로 찍어내는 **'쿠키 틀'**입니다. 이름 첫 글자를 대문자로 쓰고(예: `User`), 반드시 `new`를 붙여서 씁니다.
    
- **옵셔널 체이닝 (`?.`)**: 객체 안에 데이터가 확실히 있는지 모를 때, 에러가 나지 않게 부드럽게 넘어가 주는 마법의 물음표입니다.
    
- **가비지 컬렉션(청소부)**: 아무도 주소를 기억하지 않아 영영 찾을 수 없게 된 객체는 자바스크립트 엔진이 알아서 메모리에서 지워버립니다.

## 6. 원시값의 화려한 변신 (숫자와 문자)

숫자나 문자는 객체가 아닌 단순한 값(원시값)이지만, 필요할 때 잠깐 객체인 척하며 유용한 기능들을 빌려 씁니다.

- **숫자형 다루기**: `num.toFixed(2)`로 소수점을 자르거나, `parseInt("100px")`로 글자 속에 숨은 숫자만 뽑아낼 수 있습니다.
    
- **문자형 다루기**: `str.includes("사과")`로 단어를 찾거나, `str.slice(1, 3)`으로 원하는 부분만 싹둑 자를 수 있습니다.
    
- **주의**: `new Number()`나 `new Boolean()` 처럼 굳이 복잡하게 객체로 만들지 마세요! 버그의 원인이 됩니다.

## 7. 배열(Array)의 모든 것

배열은 순서표(인덱스)가 붙어있는 아주 유연한 리스트입니다.

- **넣고 빼기**:
    
    - `push` / `pop`: 배열의 **맨 뒤**에 넣고 뺍니다. (다른 요소를 건드리지 않아 속도가 아주 빠름!)
        
    - `unshift` / `shift`: 배열의 **맨 앞**에 넣고 뺍니다. (뒤에 있는 애들을 다 밀어야 해서 속도가 느림)
        
- **배열의 만능 칼, `splice`**: 원하는 위치를 찌르고 들어가서 요소를 삭제하고, 그 자리에 새 요소를 끼워 넣을 수 있습니다 (원본이 변함).
    
- **안전한 복사, `slice`**: 원본을 망가뜨리지 않고 원하는 만큼만 복사해서 가져옵니다.
    
- **자주 하는 실수**: 숫자가 든 배열을 정렬할 때 그냥 `sort()`를 쓰면 "15"가 "2"보다 먼저 나옵니다 (사전순 정렬). 숫자는 꼭 `sort((a,b) => a - b)`처럼 규칙을 알려줘야 해요.

## 8. 실전! 배열 메서드 체이닝 (데이터 요리하기)

실제 서비스처럼 데이터를 요리조리 다듬는 고급 스킬들입니다.

- **`filter` (거름망)**: 조건에 맞는 녀석들만 남긴 **새 배열**을 만듭니다 (예: 수많은 영수증 중 카테고리가 '식비'인 것만, 가격이 5000원 이상인 상품만 남기기).
    
- **`map` (가공기)**: 배열의 모든 요소를 가공해서 아예 **새로운 형태의 배열**로 변신시킵니다 (예: 상품 이름을 전부 대문자로 바꾸기).
    
- **`reduce` (계산기)**: 배열의 모든 값을 차곡차곡 모아 **단 하나의 결과물**(예: 지출 총합 금액)로 만들어냅니다.
    
- **체이닝 (연결하기)**: 위 메서드들을 점(`.`)으로 계속 연결하면, 원본 데이터를 망가뜨리지 않고 원하는 결과만 한 줄의 깔끔한 코드로 얻어낼 수 있습니다.
