# 옵셔널 체이닝 '?.
옵셔널 체이닝(optional chaining) `?.`을 사용하면 프로퍼티가 없는 중첩 객체를 에러 없이 안전하게 접근할 수 있습니다.

## 1. 옵셔널 체이닝의 3가지 형태 (복습)

옵셔널 체이닝은 단순히 "에러를 안 나게 하는 것"이 아니라, **"존재하지 않을 수도 있는 경로를 안전하게 탐색"**하는 도구입니다.

|**형태**|**예시**|**설명**|
|---|---|---|
|**속성 접근**|`user?.name`|`user`가 있으면 `name`을, 없으면 `undefined`를 반환|
|**대괄호 접근**|`user?.['email']`|대괄호 표기법(변수로 접근 등)에서도 안전하게 사용|
|**메서드 호출**|`user?.sayHi()`|`user`가 존재할 때만 함수를 실행 (없어도 에러 X)|

---

## 2. 반드시 알아야 할 추가 개념 (중요!)

### ① 단락 평가 (Short-circuiting)

`?.`은 왼쪽 평가 대상이 `null`이나 `undefined`이면 **즉시 실행을 멈춥니다.** 오른쪽에 있는 그 어떤 연산도 수행하지 않죠.

JavaScript

```
let user = null;
let x = 0;

user?.sayHello(x++); // user가 null이므로 실행이 여기서 멈춤!

console.log(x); // 0 (x++이 실행되지 않았음)
```

> **Tip:** 오른쪽에서 부수 효과(변수 값 변경 등)가 일어나는 코드는 작성하지 않는 것이 좋습니다.

### ② 존재하지 않는 함수 호출 시 주의사항

`user?.method()`는 `user`가 null인지는 확인해주지만, **`user`는 있는데 `method`가 함수가 아닌 경우**에는 에러가 발생합니다.

- `user?.method()` : `user`가 없으면 통과.
    
- `user.method?.()` : `user`는 있는데 `method`라는 함수가 있을 때만 호출하고 싶을 때 사용.
    

### ③ 대입 연산자 왼쪽에는 사용 불가

옵셔널 체이닝은 값을 **읽을 때**만 사용합니다. 값을 쓸 때는 사용할 수 없습니다.

- `user?.name = "Alice";` // **SyntaxError!** ❌

## 3. 실전 활용 팁: "언제 써야 할까?"

보내주신 내용처럼 `?.`은 남용하면 안 됩니다. 아래 기준을 참고해 보세요.

### ✅ 사용할 때 (Optional)

- **API 응답 데이터:** 서버에서 데이터가 올 수도 있고 안 올 수도 있는 구조일 때.
    
- **UI 상태:** 특정 조건에서만 나타나는 컴포넌트의 데이터에 접근할 때.
    

### ❌ 사용하지 말아야 할 때 (Required)

- **비즈니스 로직 필수 값:** "사용자 ID"처럼 반드시 있어야 하는 값에 `?.`을 쓰면, 에러가 나야 할 곳에서 `undefined`가 발생해 버그를 찾기 훨씬 힘들어집니다.
    
- **디버깅 시:** 어디서부터 데이터가 꼬였는지 파악해야 할 때는 차라리 에러(NPE)가 터지는 게 나을 때도 있습니다.

# 📑 객체의 원시형 변환 (Object to Primitive)

자바스크립트는 객체끼리 더하거나(`obj1 + obj2`), 객체를 출력(`alert(obj)`)할 때 **자동 형 변환**을 수행합니다. 이때 객체는 **원시값(string, number, boolean 등)**으로 변환됩니다.

---

## 1. 세 가지 타입의 'Hint'

자바스크립트 엔진은 형 변환을 할 때 "어떤 자료형으로 바꿀 것인가?"에 대한 목표치를 설정하는데, 이를 **Hint**라고 합니다.

|**Hint**|**발생 상황**|**예시**|
|---|---|---|
|**`"string"`**|문자열을 기대하는 연산|`alert(obj)`, `String(obj)`, 객체를 키로 사용|
|**`"number"`**|수학 연산을 기대하는 연산|`Number(obj)`, `-`, `*`, `/`, `>` (비교)|
|**`"default"`**|자료형을 확신할 수 없을 때|`obj1 + obj2` (더하기), `obj == 1` (동등 비교)|

> **Note:** 이항 덧셈(`+`)은 문자열 합치기일 수도, 숫자 더하기일 수도 있기 때문에 `default` hint를 사용합니다. 대부분의 내장 객체는 `default`와 `number`를 동일하게 처리합니다.

---

## 2. 형 변환 알고리즘 (우선순위)

엔진은 다음 순서에 따라 메서드를 찾고 실행합니다.

1. **`Symbol.toPrimitive(hint)`**가 있는지 확인 → 있다면 이것만 실행.
    
2. 없다면, Hint가 **`"string"`**인 경우:
    
    - `toString()` → `valueOf()` 순서로 호출.
        
3. Hint가 **`"number"`** 또는 **`"default"`**인 경우:
    
    - `valueOf()` → `toString()` 순서로 호출.
        

---

## 3. 실전 구현 예시

### ① 현대적인 방식: `Symbol.toPrimitive`

하나의 메서드로 모든 형 변환을 제어할 수 있습니다.

JavaScript

```
let user = {
  name: "John",
  money: 1000,

  [Symbol.toPrimitive](hint) {
    console.log(`hint: ${hint}`);
    return hint == "string" ? `{name: "${this.name}"}` : this.money;
  }
};

alert(user);       // hint: string -> "{name: "John"}"
console.log(+user); // hint: number -> 1000
console.log(user + 500); // hint: default -> 1500
```

### ② 전통적인 방식: `toString`과 `valueOf`

`Symbol.toPrimitive`가 없던 시절부터 사용된 방식입니다.

JavaScript

```
let user = {
  name: "John",
  money: 1000,

  // hint가 "string"일 때 우선 호출
  toString() {
    return this.name;
  },

  // hint가 "number", "default"일 때 우선 호출
  valueOf() {
    return this.money;
  }
};


let user = {
  name: "John",
  money: 1000,

  // 1. hint가 "string"인 경우 호출됨
  toString() {
    return `{name: "${this.name}"}`;
  },

  // 2. hint가 "number"나 "default"인 경우 호출됨
  valueOf() {
    return this.money;
  }
};

// --- 실행 결과 ---

alert(user);       // toString -> {name: "John"}
alert(+user);      // valueOf -> 1000 (단항 덧셈은 숫자형 변환 유도)
alert(user + 500); // valueOf -> 1500 (이항 덧셈은 default hint 사용)
```

---

## 4. 추가 꿀팁 및 주의사항 (심화)

### ⚠️ 메서드가 원시값이 아닌 객체를 반환하면?

- **`Symbol.toPrimitive`**: 반드시 원시값을 반환해야 합니다. 객체를 반환하면 **TypeError**가 발생합니다.
    
- **`toString` / `valueOf`**: 객체를 반환해도 에러는 안 나지만, 반환값이 무시됩니다 (메서드가 없는 것처럼 동작).
    

### 🔄 변환 후 재변환 (Double Conversion)

객체가 원시값으로 변환된 후에도, 연산자가 요구하는 타입이 아니라면 **한 번 더 형 변환**이 일어날 수 있습니다.

JavaScript

```
let obj = {
  toString() { return "2"; }
};

console.log(obj * 2); 
// 1단계: obj가 문자열 "2"로 변환 (hint: number)
// 2단계: 곱셈을 위해 "2"가 숫자 2로 재변환됨
// 결과: 4
```

### 💡 왜 `valueOf`는 이름이 이런가요?

자바의 `valueOf()`와 비슷하게 "객체가 가진 가공되지 않은 값"을 반환하라는 의미입니다. 자바스크립트 초기 설계에서 객체는 본질적으로 '값'보다는 '참조'였기에, 계산을 위한 '원시 값'을 따로 뽑아내는 함수가 필요했던 것입니다.

---

## 5. 요약

1. 자바스크립트는 연산 시 객체를 자동으로 원시형으로 바꾼다.
    
2. **Hint**(`string`, `number`, `default`)에 따라 호출되는 메서드 순서가 다르다.
    
3. 가장 강력한 제어권은 **`Symbol.toPrimitive`**에 있다.
    
4. 모든 형 변환 메서드는 결국 **원시값**을 반환해야 의미가 있다.
