## 1. 실무 환경 (Testing & Polyfill)
현대적인 자바스크립트 개발은 "그냥 짜는 것"이 아니라 "검증하며 짜는 것"입니다.

- **Mocha & Chai**: 
    - **Mocha**: 테스트를 실행하는 프레임워크 (`describe`, `it`).
    - **Chai**: 결과를 비교하는 라이브러리 (`assert.equal`).
- **폴리필(Polyfill)**: 최신 자바스크립트 문법을 지원하지 않는 구식 브라우저(IE 등)를 위해, 엔진이 해석할 수 있도록 기능을 '채워주는' 코드 조각입니다.
- **닌자 코드**: 남이 읽기 힘들게 꼬아놓은 나쁜 코드를 뜻하며, 절대 지양해야 합니다.

## 2. 객체(Object) 기초
객체는 데이터를 '키(Key): 값(Value)'의 쌍으로 저장하는 바구니입니다.

### ✅ 객체 조작법
- **추가/수정**: `obj.prop = value` (점 표기법)
- **삭제**: `delete obj.prop`
- **확인**: `"key" in obj` (해당 키가 있는지 확인)
- **순회**: `for (let key in obj)` (객체의 모든 키를 한 바퀴 돎)

> **프로퍼티 이름 제약**: 키 이름에 공백이 포함되면 `"likes birds": true`처럼 따옴표로 감싸야 하며, 접근할 때는 `obj["likes birds"]` 형식을 써야 합니다.

## 3. 참조(Reference)에 의한 복사 ⭐⭐⭐
이 부분은 자바스크립트에서 가장 중요한 개념 중 하나입니다.

- **변수에는 '주소'가 저장됩니다**: 객체를 변수에 할당하면 객체 자체가 아니라, 객체가 저장된 **메모리상의 주소(참조값)**가 복사됩니다.
- **부작용**: `let user2 = user;`라고 복사하면 두 변수는 **같은 객체**를 가리킵니다. `user2`를 수정하면 `user`도 변합니다.

## 4. 가비지 컬렉션 (Garbage Collection)
자바스크립트 엔진은 사용하지 않는 메모리를 자동으로 청소합니다.

- **도달 가능성(Reachability)**: 루트(전역 변수 등)에서부터 참조를 타고 가서 닿을 수 있는 값은 메모리에 유지됩니다.
- **삭제 대상**: 어디에서도 참조되지 않아 '도달할 수 없는' 객체는 엔진이 알아서 메모리에서 지워버립니다.

## 5. 실습 코드 개선 (숫자 프로퍼티 2배 만들기)

질문하셨던 코드를 매개변수를 사용하여 재사용 가능하게 수정한 최종본입니다.

```javascript
let menu = {
  width: 200,
  height: 300,
  title: "My menu"
};

/**
 * 객체 내의 숫자 타입 값만 2배로 곱해주는 함수
 * @param {Object} obj - 수정할 객체
 */
function multiplyNumeric(obj) {
  for (let key in obj) {
    // 1. 타입을 확인하여 숫자일 때만 실행
    if (typeof obj[key] === 'number') {
      // 2. 참조를 통해 원본 객체의 값을 직접 수정
      obj[key] *= 2;
    }
  }
}

// 함수 실행
multiplyNumeric(menu);

console.log(menu); 
// 결과: { width: 400, height: 600, title: "My menu" }
